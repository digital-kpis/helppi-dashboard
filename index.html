🗄️ Passo a passo — criar o banco
1
Criar conta/projeto no Supabase
Acesse supabase.com, crie uma conta grátis e um novo projeto (escolha uma senha de banco e uma região próxima).
Criar projeto →
2
Criar a tabela
No menu lateral, abra SQL Editor, cole o código abaixo e clique em Run:
create table helppi_dashboard (
  id int primary key,
  data jsonb not null,
  updated_at timestamptz not null default now()
);
alter table helppi_dashboard enable row level security;
create policy "helppi_anon_all" on helppi_dashboard
  for all using (true) with check (true);
💡 A policy libera leitura/escrita para o time todo — ideal pra uso interno. Se quiser mais restrição, ajuste depois no Supabase.
3
Pegar URL e chave do projeto
Vá em Project Settings → API. Copie o Project URL e a chave anon public.
4
Preencher configurações ao lado →
Cole a URL e a chave no painel ao lado, clique em Testar conexão e depois em ☁️ Salvar agora.

🐙 Passo a passo — criar link de acesso
1
Criar conta no GitHub
Acesse github.com/signup e crie uma conta gratuita. Se já tiver, pule este passo.
Criar conta →
2
Criar repositório público
Clique no link abaixo, preencha o nome do repositório (ex: helppi-dashboard), marque Public e clique em Create repository.
Criar repositório →
3
Gerar Token de Acesso
Acesse Settings → Tokens. Em Note escreva helppi, em Expiration escolha No expiration, marque o escopo repo e clique em Generate token. Copie o token gerado (ele só aparece uma vez).
Gerar token →
4
Ativar GitHub Pages
No repositório criado, vá em Settings → Pages, em Branch selecione main e clique em Save. Aguarde ~1 minuto e seu link estará disponível como:
https://SEU-USUARIO.github.io/helppi-dashboard/
💡 Qualquer pessoa com o link poderá visualizar o dashboard sem precisar de conta.
5
Preencher configurações ao lado →
Cole seu usuário, nome do repositório e token no painel ao lado. Clique em Testar conexão para validar e depois em Publicar dashboard.
